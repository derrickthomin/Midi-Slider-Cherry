1. Overview

The LumaFader (also known as DJBB Cherry Faders) is a compact, open-source MIDI controller built on a Raspberry Pi Pico W running CircuitPython. It packs an enormous amount of control into a palm-sized package:

  • 4 long-throw analog faders (Sliders A–D)
  • 4 mechanical buttons (Cherry MX-compatible) with integrated RGB LEDs
  • 69 NeoPixel RGB LEDs — 16 per slider column + 4 button LEDs + 1 mode indicator
  • USB-C for power, USB MIDI, and serial configuration
  • TRS-A MIDI output (3.5mm jack) for hardware synths & gear
  • Control up to 68 unique MIDI parameters across 4 banks × 4 pages + a global bank

3. Quick Start

  1. Connect USB-C to your computer. The device powers on with a rainbow startup animation.
  2. Your DAW or MIDI software should see a device named "LumaFader" as a MIDI input.
  3. Move the sliders — they immediately send MIDI CC messages on the global bank (CC 0–3, Channel 1 by default).
  4. Hold a button to temporarily switch to a bank. The LED column colors change to show which bank is active.
  5. Release the button to return to the global bank.
Tip
Out of the box, the LumaFader sends CC 0–67 across its 4 pages × 4 banks — enough to control an entire mix. No configuration required.

TRS MIDI

All MIDI messages are sent simultaneously over both USB and the TRS-A jack. Just plug a 3.5mm TRS-to-DIN adapter (Type A) into your synth — no setup needed.

4. Banks & Pages

Banks and pages are how LumaFader maps 4 physical sliders to up to 68 different MIDI parameters.

Global Bank

When no button is held or locked, the sliders send on the global bank — the default CC numbers (0–3) and channel. LED color: warm gold.

Banks 1–4

Each button corresponds to a bank. Hold a button to temporarily activate that bank — sliders switch to that bank's CC numbers, channel, and message type. Release to return to global.

Locking a Bank

Double-press a button to lock that bank. It stays active even after you release. The button LED stays lit. Double-press the same button to unlock.

Pages 1–4

Each page contains its own set of 4 banks with independent CC mappings. This gives you 4 × 4 × 4 = 64 unique slider assignments plus 4 more on the global bank.

ActionHow
Next PageHold Button 1 (bottom) + tap Button 4 (top)
Previous PageHold Button 4 (top) + tap Button 1 (bottom)

The current page is shown by a white LED on the corresponding button (Button 1 = Page 1, etc.).

Channel Inheritance
Each level can override the MIDI channel. The first non-empty value wins:
Bank Channel → Page Channel → Global Channel

5. Pickup Mode & Jump Mode

Pickup Mode (Default)

When you switch banks, each slider's physical position likely won't match the last value sent for its new CC assignment. To prevent jarring parameter jumps, LumaFader uses pickup mode:

  • After a bank change, each slider is "disengaged" until it physically crosses the last-sent value for its new CC number.
  • Once the slider passes through the target value, it "picks up" and sends freely.
  • The LED column shows the last-sent value (the target you need to cross), not the raw slider position — so you always know which direction to move.
Tip
If a slider is already very close to the target value (within 2 CC units) or at the extreme ends (0 or 127), it picks up automatically.

Jump Mode

Sometimes you want instant control without waiting for pickup. Jump mode disables pickup and sends the slider's current position immediately, regardless of the last-sent value.

Toggle Jump ModeHold Button 2 + Button 3 (both middle) then tap Button 1 or Button 4
Indicator LEDOrange = Jump Mode ON  ·  Green = Normal (Pickup) Mode

7. Web Configuration

LumaFader has a built-in web configuration interface that runs in Chrome or Edge (any browser supporting the Web Serial API).

Accessing the Configurator

  1. Connect LumaFader via USB-C.
  2. Open the web_config.html file in Chrome or Edge.
  3. Click "Connect" and select the LumaFader serial port from the browser prompt.
  4. The interface shows a live representation of your device — sliders, buttons, and all settings.
Browser Requirement
Web Serial API is only available in Chromium-based browsers (Chrome, Edge, Opera). Safari and Firefox are not supported.

What You Can Configure

  • CC Numbers — click on any CC number field and type a new value (0–127)
  • MIDI Channels — set per-bank, per-page, or global channel (1–16, or multi-channel like 1|2|3)
  • Message Type — choose CC (Control Change) or AT (Channel Aftertouch) per bank, page, or globally
  • MIDI Learn — click "Learn" on a slider, then send a CC from your DAW or another controller — LumaFader catches it and auto-assigns the CC number
  • Live monitoring — the web UI updates in real-time as you move sliders and press buttons on the physical device

MIDI Learn

  1. Click the "Learn" button next to a slider.
  2. Send a MIDI CC message from your DAW or another controller (via USB or TRS).
  3. LumaFader captures the CC number (and optionally the channel) and assigns it.
  4. Learn mode times out after 30 seconds if no message is received.

Saving Settings

After making changes, click "Save". Settings are written to the device's internal storage. The device must be restarted for changes to take effect (unplug and replug USB, or use the reset button).

9. Advanced Topics

USB Drive Mode

For manual firmware updates or editing settings.json directly:

  1. Unplug the LumaFader.
  2. Hold all 4 buttons while plugging in USB-C.
  3. The device mounts as a USB drive named "LUMAFADER".
  4. The LEDs will blink red 3 times to confirm drive mode is active.
  5. Edit files as needed, then eject and unplug to return to normal mode.
Warning
In USB drive mode, the device's filesystem is read-only to the firmware — settings changes via the web configurator will not save. Always eject properly before unplugging.

Multi-Channel Output

Any channel setting accepts a pipe-separated list of channels. For example, setting the global channel to "1|2|10" will send every CC message to channels 1, 2, and 10 simultaneously.

Channel Aftertouch Mode

Setting the message type to "AT" sends Channel Pressure (Aftertouch) messages instead of CC. In this mode, CC numbers are ignored — only one pressure value is sent per channel. This is useful for controlling synth parameters that respond to aftertouch.

Multi-Bank Mode

Holding multiple buttons simultaneously activates multi-bank mode. The first button pressed is the "primary" bank and controls the slider CC assignments. Additional held buttons add their CC assignments as secondary outputs — each slider sends multiple CC messages per movement.

Adaptive Smoothing

The sliders use a state-machine-based adaptive smoothing algorithm:

  • CHANGING state: Low threshold (1 CC unit), fast response — triggers on any small movement
  • STABLE state: Higher threshold (3 CC units), filters noise — activates after 1 second of no movement

This means the sliders are responsive when you're actively adjusting them, but resistant to electrical noise when idle.

Firmware & Bootloader

The LumaFader runs CircuitPython 8.2.x on a Raspberry Pi Pico W (RP2040). The processor is overclocked to 270 MHz for smooth LED rendering and fast MIDI response.

To access the RP2040 bootloader for flashing new CircuitPython firmware: hold the small BOOTSEL button on the PCB while plugging in USB.

10. Troubleshooting

SymptomSolution
LEDs blink red 3× on startup The device is in USB drive mode (all 4 buttons were held during boot). Unplug and replug normally.
Slider doesn't send MIDI after switching banks Pickup mode is active. Move the slider until it crosses the target value (shown by the LED fill level). Or enable Jump Mode.
Web configurator won't connect Ensure you're using Chrome or Edge. Check that no other application is holding the serial port (close other serial monitors). Try a different USB cable.
Settings don't save from web config Make sure the device is not in USB drive mode (no red blinks at startup). Settings save to internal storage and require a restart to take effect.
DAW doesn't see MIDI input Unplug and replug USB. On macOS, check Audio MIDI Setup. On Windows, check Device Manager for the MIDI device. Ensure the device is not in USB drive mode.
TRS MIDI not working Ensure you're using a TRS Type-A adapter (not Type-B). The TRS jack is output only — it mirrors all USB MIDI output.
Page indicator blinks instead of changing You've reached the page limit (1–4). Pages don't wrap around.
Slider LEDs don't match physical position This is by design — LEDs show the last CC value sent, not the raw slider position. This helps with pickup mode. Move the slider to match the LED level.
